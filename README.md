# Задача доставки заказов

**1.	Постановка задачи**

В логистическом центре имеются заказы, каждый из которых необходимо доставить в свой пункт назначения. По каждому из заказов известны следующие данные:
- Директивный срок – время, не позже которого заказа должен быть доставлен.
- Время, требуемое для перемещения из логистического центра в пункты назначения
- Время, требуемое для перемещения между пунктами назначения.

Требуется определить такой порядок доставки заказов, при котором число заказов с нарушенным директивным сроком будет минимальным.
  
**2.	Метод ветвей и границ (BranchAndBound)**

Процесс начинается с корневой вершины, представляющей начальное состояние. На каждой итерации для всех листьев рассчитываются нижняя и верхняя оценки.

Далее алгоритм выполняет две операции:
- Ветвление - выбирается наиболее перспективная вершина и генерируются все возможные следующие шаги.
- Отсев - ветви, у которых нижняя оценка хуже текущего лучшего найденного результата, отбрасываются.

Алгоритм работает до тех пор, пока не будет найдена вершина, для которой верхняя и нижняя оценки совпадут.

**3.	Базовая стратегия ветвление (OptimisticBranchChoosing)**

Следующим для рассмотрения выбирается узел дерева с самой маленькой нижней оценкой.
  
**4.	Базовая верхняя оценка (UpBoundFinder)**

Определяется как суммарное количество нарушений директивных сроков, если каждый раз мы будем идти в ближайший по времени пункт назначения, в котором еще не был нарушен директивный срок.
  
**5.	Базовая нижняя оценка (LowBoundFinder)**

Определяется как суммарное количество нарушений директивных сроков, если будем считать, что из рассматриваемого пункта во все оставшиеся мы пойдем параллельно.
  
**6.	Предложенная стратегия ветвления (MyChoosingStratagy)**

Следующим для рассмотрения выбирается узел дерева, у которого минимально произведения верхней и нижней оценок.
  
**7.	Предложенная верхняя оценка (MyUpBoundFinder)**

Достижимая оценка определяется как суммарное количество нарушений директивных сроков, если каждый раз мы будем идти в пункт назначения, в котором срок доставки ближе всего к нарушению, но еще не нарушен.

**8.	Предложенная нижняя оценка (MyLowFinder)**

Нижняя оценка для текущей вершины определяется как минимум из базовых нижних оценок всех смежных вершин, где каждая базовая оценка строится с учетом шага на глубину 2. 
решение быстрее.

**9.	Подсчет количества обработанных вершин (NodeCountDecorator)**

Реализуется через декоратор, который увеличивает кол-во вершин на 1 при каждом подсчете верхней оценки.
